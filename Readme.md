# 🏥 MedAssistant — RAG-чат-бот для медицинских учреждений

> Интеллектуальный чат-бот на основе локальной базы знаний с векторным поиском (RAG).  
> Отвечает на вопросы пользователей по документам медицинских учреждений — расписаниям, услугам, контактам и т.д.  
> Реализован в рамках преддипломной практики. Автор — [@Muxamor1](https://t.me/Muxamor1)

### Шаг 1 — Добавить документы

Положите файлы с информацией о медучреждениях в папку `docs/`.  
Можно создавать подпапки — они автоматически становятся категориями:

```
docs/
├── Поликлиника №1/
│   ├── расписание_врачей.xlsx
│   └── услуги_и_цены.docx
├── Поликлиника №2/
│   └── контакты.pdf
└── общая_информация.docx
```

Поддерживаемые форматы:

| Формат | Особенности |
|--------|-------------|
| `.docx` | Word-документы, таблицы с объединёнными ячейками |
| `.xlsx` / `.xls` | Excel-таблицы: справочники, расписания специалистов |
| `.pdf` | PDF-документы любой структуры |
| `.doc` | Старый формат Word (только Windows) |

### Шаг 2 — Запустить сервер

Дважды кликните на `server.exe` или запустите через терминал:

```bash
server.exe
```

При первом запуске сервер проиндексирует все документы из `docs/` — это займёт несколько минут.  
Когда увидите эту строку — сервер готов:

```
✅ Сервер готов
 * Running on http://0.0.0.0:5000
```

### Шаг 3 — Открыть интерфейс

Откройте файл `chat_widget.html` в браузере и задавайте вопросы боту.

## 📂 Обновление базы знаний

Для добавления, изменения или удаления документов:

1. Внесите изменения в папку `docs/`
2. Перезапустите `server.exe`

Сервер автоматически определит изменившиеся файлы и обновит только их — полная переиндексация не нужна.

Для принудительной полной переиндексации запустите через терминал:

```bash
server.exe --reindex
```

## 🌐 Настройка IP и порта

По умолчанию сервер запускается на `0.0.0.0:5000` (доступен в локальной сети).  
Можно изменить через аргументы командной строки:

```bash
# По умолчанию (доступен в локальной сети)
server.exe

# Только на вашем компьютере (никто в сети не достучится)
server.exe --host 127.0.0.1

# Конкретный IP вашей машины в сети
server.exe --host 192.168.1.100

# Другой порт
server.exe --host 0.0.0.0 --port 8080

# Комбинация
server.exe --host 192.168.1.100 --port 8080
```

Узнать свой IP в локальной сети:
```bash
ipconfig
```
Ищите строку **IPv4-адрес** — обычно вида `192.168.x.x`.

## 🔌 API

Сервер предоставляет REST API на порту `5000`. Можно интегрировать в любой сайт или приложение.

### `POST /ask` — задать вопрос боту

**Запрос:**
```json
{
  "query": "Как записаться на приём к терапевту?"
}
```

**Ответ:**
```json
{
  "answer": "Для записи к терапевту позвоните по номеру +375 (xx) xxx-xx-xx..."
}
```

### `POST /debug` — вопрос с указанием источников

**Запрос:**
```json
{
  "query": "Какой режим работы регистратуры?"
}
```

**Ответ:**
```json
{
  "answer": "Регистратура работает с 8:00 до 20:00 в будние дни...",
  "sources": ["расписание.docx", "контакты.xlsx"]
}
```

### `GET /health` — проверка состояния сервера

**Ответ:**
```json
{
  "status": "ok"
}
```

### Пример интеграции (JavaScript)

```javascript
const response = await fetch('http://localhost:5000/ask', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ query: 'Ваш вопрос здесь' })
});

const data = await response.json();
console.log(data.answer);
```

## ❓ Решение проблем

**Сервер запустился, но бот не отвечает в браузере**  
→ Убедитесь, что в консоли `server.exe` нет красных ошибок. Проверьте, что порт 5000 свободен:
```bash
netstat -ano | findstr :5000
```

**Бот отвечает "не могу найти информацию" на все вопросы**  
→ Папка `docs/` пуста, либо индекс ещё не создан. Добавьте документы и перезапустите сервер.

**Сервер долго запускается (2–5 минут)**  
→ Это нормально при первом запуске. Загружается модель эмбеддингов весом ~2 ГБ.

**Антивирус блокирует или удаляет `server.exe`**  
→ Ложное срабатывание на Nuitka-сборку (скомпилированный Python часто триггерит эвристику). Добавьте файл в исключения вашего антивируса.

**Ошибка `CUDA out of memory`**  
→ Бот работает полностью на CPU, CUDA не используется. Если видите эту ошибку — напишите автору.
